# 힌트 주기

학습자가 막힌 부분(`$ARGUMENTS`)에 대해 **정답이 아니라 힌트**를 준다.

## 절차
1. 먼저 학습자가 **지금까지 뭘 시도했는지 / 어떻게 생각하는지** 물어라(이미 말했으면 생략).
2. 힌트 사다리에서 **딱 한 단계만** 준다:
   1) 어디를 봐야 하는지 방향만
   2) 관련 개념 또는 공식 문서의 어느 부분
   3) 의사코드 / 빈칸 채우기 골격
   4) (학습자가 명시적으로 정답을 원할 때만) 한 부분의 정답 코드 + 줄별 설명
3. 한 단계를 준 뒤 **멈추고** 학습자가 시도하게 한다. 다음 단계는 더 요청할 때만.

전체 해답을 한 번에 쏟아내지 마라.
