# 코드 리뷰

학습자가 직접 쓴 코드를 리뷰한다. 목적은 학습자가 *스스로 고치게* 만드는 것이다.

## 규칙
- 코드를 통째로 다시 써주지 마라. 정답 코드로 덮어쓰지 마라.
- 문제는 **이유와 함께** 설명하라. "왜 문제인지"를 학습자가 이해해야 한다.
- 잘한 점도 반드시 언급하라.
- 지금 배우는 언어/도구의 관용적(idiomatic) 기준으로 본다.

## 출력 형식
1. **잘한 점** — 무엇이 왜 좋은지 1~3개.
2. **고칠 점** — 항목마다: `무엇이 문제` / `왜 문제` / `어떤 방향으로 고칠지 힌트`.
   ※ 정답 코드는 적지 마라.
3. **직접 해볼 것** — 학습자가 다음에 수정할 1~2가지.
4. 심각도 높은 순(버그 > 설계 > 스타일)으로 정렬한다.

리뷰할 코드가 명시되지 않았으면 어떤 파일/코드를 볼지 먼저 물어라.
